# Installation

## Prerequisites

- Vue 3.5+
- Tailwind CSS v4
- Node.js 18+

## Install

```bash
npm install ocelot-ui
```

## Setup

### 1. Configure Tailwind to scan the library

Ocelot UI components use Tailwind utility classes. Add the compiled library to your Tailwind CSS source scanning so the utilities are included in your build. Also add the dark mode variant so `.dark` class toggling works:

```css
/* your main CSS file */
@source "../node_modules/ocelot-ui/dist/ocelot-ui.es.js";
@custom-variant dark (&:where(.dark, .dark *));
@import "tailwindcss";
```

### 2. Import the library stylesheet

The library ships a small additional stylesheet for dark mode variant configuration and Vue transitions. Import it once in your app entry point or root CSS file:

```js
// main.js / main.ts
import 'ocelot-ui/style.css'
```

Or in your root CSS:

```css
@import 'ocelot-ui/style.css';
```

### 3. Register components

```js
// main.js / main.ts
import { createApp } from 'vue'
import OcelotUI from 'ocelot-ui'
import 'ocelot-ui/style.css'
import App from './App.vue'

createApp(App).use(OcelotUI).mount('#app')
```

Or import individually:

```js
import OcelotUI from 'ocelot-ui'
const { Button, Input, Dialog } = OcelotUI
```

---

## Dark mode

Dark mode is toggled via the `.dark` class on a parent element (typically `<html>`):

```html
<html class="dark">
  ...
</html>
```

---

## Dialog portal target

The `Dialog` component uses Vue's `<Teleport>` to render outside the component tree.
Add a `#portal-target` element to your `index.html`:

```html
<div id="app"></div>
<div id="portal-target"></div>
```
